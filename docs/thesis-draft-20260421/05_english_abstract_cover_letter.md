## 3. English Abstract

Block-based programming environments reduce the entry barrier for novice learners, but they often fail to support a smooth transition to authentic text-based programming. In contrast, text-based Python environments provide practical relevance and ecosystem scalability, yet they impose substantial overhead in installation, configuration, package management, and syntax debugging. This dissertation presents an educational multi-representation programming system that integrates bidirectional block-to-text transformation, an in-browser Python interpreter, JavaScript-based interaction bridges, AI-assisted library extension, and learning analytics. The proposed system allows learners to author programs in Blockly or Python while maintaining a shared semantic structure across representations. Programs created in either modality converge to a unified Python execution pipeline running inside the browser through a WebAssembly-based runtime. The system further connects Python execution to interactive stage behaviors through a browser-side event bridge and supports extensible library packages that combine block definitions, code generators, and runtime bindings. The dissertation evaluates the system at two levels: system-level performance, including execution fidelity, responsiveness, package support, and extensibility; and educational effectiveness, including programming achievement, block-to-text transition ability, self-efficacy, cognitive load, usability, code quality, and usage behavior. The main contribution of this work is a unified programming environment that repositions block-based and text-based programming not as separate stages, but as coordinated representations over a shared execution semantics for introductory programming education.

## 5. Short 150-Word Version

This dissertation proposes an educational programming system that integrates Blockly, bidirectional block-to-text transformation, and an in-browser Python runtime. Unlike conventional tools that either support visual programming or text-based programming in isolation, the proposed system maintains semantic continuity between both representations and executes them through a unified Python pipeline inside the browser. The system combines a WebAssembly-based interpreter, JavaScript interaction bridges, AI-assisted library generation, and detailed learning analytics. It is designed to reduce setup friction, support smooth transition from blocks to Python, and provide a scalable platform for interactive programming instruction. The evaluation plan covers both system performance and educational effectiveness, including execution fidelity, transition performance, self-efficacy, cognitive load, usability, and code quality.
