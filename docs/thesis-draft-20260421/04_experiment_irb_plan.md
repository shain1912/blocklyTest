# 실험 설계 및 IRB 가안

## 1. 연구 목적

본 연구의 목적은 양방향 블록-텍스트 변환과 브라우저 내 Python 인터프리터를 결합한 교육용 다중표현 프로그래밍 시스템이 학습자의 Python 전환 성과, 코드 품질, 자기효능감, 인지부하, 사용성에 미치는 영향을 검증하는 것이다. 추가로 AI 보조 기능의 교육적 효용과 과의존 위험을 함께 분석한다.

## 2. 권장 연구설계

### 2.1 기본 설계

`2 x 2 요인설계`

- 요인 1: 표현 환경
  - `text-only`
  - `dual-modality`
- 요인 2: AI 지원
  - `off`
  - `on`

총 4집단.

### 2.2 대안 설계

만약 표본 확보가 어렵다면 아래 3집단 설계로 축소할 수 있다.

- blocks-first then text
- text-only
- dual-modality

하지만 논문 힘은 `2 x 2`가 더 세다.

## 3. 참여자

### 3.1 추천 대상

- 부산대학교 학부 `Python 기초`, `컴퓨팅사고`, `프로그래밍입문`, `CS1` 수강생
- 비전공 교양 Python 수업도 가능하나, 논문 일관성은 전공 기초과목이 더 좋다

### 3.2 권장 표본

- 목표 표본: `N = 160`
- 최소 운영 가능 표본: `N = 120`
- 4집단이면 각 집단 30명 이상 확보를 권장

### 3.3 포함 기준

- 본 수업 수강생
- 연구 설명문을 읽고 동의한 학생
- 최소 1회 이상 사전·사후 평가에 참여한 학생

### 3.4 제외 기준

- 연구 참여 동의 철회
- 로그 수집 거부
- 사전/사후 둘 중 하나라도 완전 누락

## 4. 중재 절차

### 4.1 기간

- 권장: 4주에서 8주
- 최소: 3회차 이상의 실습 세션

### 4.2 공통 학습 내용

- 순차
- 변수
- 조건문
- 반복문
- 함수
- 이벤트 기반 상호작용
- 간단한 라이브러리 사용

### 4.3 집단별 차이

- `text-only`: Python 편집기만 사용
- `dual-modality`: 블록과 Python 양쪽 사용 가능, 실시간 전환 가능
- `AI on`: 설명형 AI, 라이브러리 보조, 오류 설명 허용
- `AI off`: 동일 과제이나 AI 사용 차단

## 5. 측정 시점

- T0: 사전설문 및 사전지식검사
- T1: 중간 로그 수집
- T2: 사후지식검사, 전환과제, 디버깅과제, 설문
- T3: 2주 또는 4주 후 지연사후검사

## 6. 종속변수와 측정도구

## 6.1 학습성과

- Python 개념검사
- 코드 추적 문제
- 오류 수정 문제
- 작은 구현 과제

평가 항목:

- 정답률
- 완성 시간
- 오류 수
- 요구사항 충족률

## 6.2 전환 성과

- 블록으로 제시된 프로그램을 Python으로 옮기기
- Python으로 제시된 프로그램을 블록으로 복원하기
- 두 표현 사이 대응 설명하기

평가 항목:

- 구문 정확도
- 의미 보존 정도
- 전환 시간
- 수정 횟수

## 6.3 자기효능감

권장 도구:

- Ramalingam and Wiedenbeck의 프로그래밍 자기효능감 척도 계열

운영 방안:

- 원문 32문항을 그대로 쓰거나
- 최근 CS1 맥락에 맞게 언어 비종속형으로 다듬은 축약형 사용

주의:

- 언어가 C++ 기준이므로 Python 맥락에 맞는 번안 절차를 문서화해야 한다.

## 6.4 인지부하

## 9. 통계 분석 계획

### 9.1 기본 비교

- 집단 간 사후점수 비교: ANCOVA
- 사전점수는 공변량 처리

### 9.2 반복측정

- 사전/사후/지연사후: linear mixed-effects model

### 9.3 행태 로그 분석

- 전환 횟수와 성과의 관계: 회귀분석
- 오류 시퀀스: sequence analysis
- AI 사용 목적별 차이: chi-square 또는 multinomial regression

### 9.4 보고 항목

- 평균
- 표준편차
- 효과크기
- 95% 신뢰구간
- 유의확률

## 10. IRB 핵심 포인트

### 10.1 연구 유형

- 최소위험 인간대상연구
- 교육 현장 기반 실험

### 10.2 예상 위험

- 학습 중 피로감
- 설문 응답 부담
- 로그 수집에 대한 심리적 부담

### 10.3 위험 완화

- 성적과 연구참여 분리
- 동의 거부 시 불이익 없음
- 익명화된 ID 사용
- 원시 로그에서 이름, 학번 제거
- AI 프롬프트 내 개인정보 마스킹

### 10.4 참여자 권리

- 언제든 철회 가능
- 철회해도 성적, 수업 참여, 지도와 무관
- 대체 과제 제공 가능

### 10.5 데이터 보관

- 암호화 저장
- 식별정보와 연구데이터 분리 보관
- 보관 기간 `[예: 3년]`
- 종료 후 파기 절차 명시

## 11. IRB 신청서용 요약문 가안

본 연구는 대학 프로그래밍 수업에서 사용되는 웹 기반 다중표현 프로그래밍 시스템의 교육 효과를 검증하기 위한 최소위험 연구이다. 참여자는 블록 기반 표현과 Python 텍스트 표현을 전환할 수 있는 학습환경 또는 비교 환경을 사용하여 정해진 프로그래밍 과제를 수행한다. 연구자는 사전/사후 지식검사, 설문, 수행 결과, 시스템 로그를 수집하여 학습성과와 사용행태를 분석한다. 수집되는 로그에는 개인 식별정보를 포함하지 않으며, 참여자는 언제든지 연구 참여를 철회할 수 있다. 연구 참여 여부는 수업 성적과 무관하며, 비참여자를 위한 대체 절차를 제공한다.

## 12. 동의서 핵심 문구 가안

- 본 연구는 프로그래밍 학습 도구의 교육적 효과를 알아보기 위한 연구입니다.
- 귀하는 수업 중 또는 별도 지정된 시간에 프로그래밍 과제, 설문, 지식검사에 참여하게 됩니다.
- 시스템 사용 과정에서 익명화된 로그 데이터가 수집될 수 있습니다.
- 연구 참여는 자발적이며, 참여하지 않거나 중도 철회하더라도 어떠한 불이익도 없습니다.
- 수집된 데이터는 연구 목적으로만 사용되며, 개인을 식별할 수 없도록 처리됩니다.

## 13. 실제로 꼭 측정해야 하는 것

지도교수 설득력과 저널 어셉트를 동시에 생각하면, 아래 여섯 개는 반드시 있어야 한다.

1. 객관식 또는 수행형 사전/사후 지식검사
2. 블록-텍스트 전환 과제
3. 디버깅 과제
4. 실행 로그
5. 자기효능감
6. 인지부하

이 여섯 개가 없으면 논문이 `도구 소개` 수준으로 보일 가능성이 높다.
